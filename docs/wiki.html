YAHA Waters IMS Philosophy

The Triton series and media-filtration systems manufactured by YAHA Water Systems are custom-configured products. Traditional systems bottle-neck procurement and inventory updates because they rely on single-person data entry (Rakesh's Excel updates).

This IMS is built on four core design pillars to solve these organizational bottlenecks:

1. Rakesh-Proofing

Capture inventory reality even when processes are bypassed. Nullable project PO links and informal tags ensure un-PO'd gate receipts are cataloged immediately rather than delayed.

2. Decoupled Action

Prem (Planning) generates BOMs, Sanoj (Store) scans gate arrivals, and shop floor workers consume parts. Stock updates automatically as a side effect of everyone doing their own step.

3. Immutable Ledger

On-hand counts are never edited directly. They are dynamically aggregated from the signed stock_movements transaction history. Every edit is an audited adjustment.

4. QR sticker design

Stickers display only the Component Code + Name + QR. Quantities are never printed. Staff scan the QR code to read the live system quantity, preventing stale paper-based counts.

Core Entity Lifecycle

Customer Order & Variant Selection

Order details and parameters (Inlet, Micron) trigger variant-driven BOM templates.

Shortfall Analysis

BOM instantiated, compared with current on-hand lots, and indents are created for missing components.

Procurement & Batching

POs raised (supports informal phone flags). Multi-project batching lines are back-fillable.

Challan GRN & QR Entry

Gate logs receipt, creates inventory lot records, and prints stickers. Unmatched items captured as untagged.

Scan Consumption

Shop floor scans QR code to consume lot items directly into project, automatically updating costs.

Role & RLS Policies

Supabase Postgres policies restrict data reads and writes per profile role. Selecting a role from the header filters the matrix.

Table / View Admin Founder Team Lead Team Member
profiles & teams All Writes Read Only None None
categories & components Full Access Read Only Read Only Read (Masked)
boms & bom_lines Full Access Read / Approve Team Write Read (Team)
purchase_orders & po_lines Full Access Read Only Team Write None
grns & grn_lines Full Access Read Only Write (Team) Insert-Only
stock_movements Read Only Read Only Insert-Only Insert-Only
Critical Immutability Rule To preserve audit ledger integrity, stock_movements has **NO UPDATE** or **NO DELETE** rules configured for any role, including Admin. All inventory adjustments, corrections, and issues must be posted as new movements.
